About

Built by a DevOps engineer
who got paged at 3am

Smart Deploy Gate exists because "it worked on Friday afternoon" is not a deployment strategy. We built ML that learns when your team ships safely — and when they don't.

The Problem

Every engineering team has the same story. A deploy goes out Friday at 5pm. The diff was "small." The tests passed. By Saturday morning, production is down.

The signals were all there — high recent failure rate, end-of-day timing, rushed commit. But nobody was reading them. Now a machine does.

The Solution

A Random Forest model that scores every deploy before it runs. Trained on your team's own build history. Gets smarter every night. Integrates with Jenkins in one pipeline stage.

No SaaS subscription. No data leaving your infrastructure. No ML expertise required. Just add the stage, get the score, ship with confidence.

The ML Engine

How the model works

Algorithm
Random Forest Classifier — 100 trees, depth 8, balanced class weights
Precision
85.1% on base model — above industry average
AUC-ROC
0.934 — near-perfect risk separation
Imbalance fix
SMOTE oversampling — prevents "always safe" predictions
Retraining
Nightly cron at 2am UTC — 5-check validation gate before swap
Drift detection
Weekly KS-test on 10 features — alerts and forces retrain if detected
